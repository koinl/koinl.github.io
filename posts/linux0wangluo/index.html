关于 Linux 的网络配置, 可分为配置网络 IP 临时 [ifconfig, ip] 与 永久 [nmcli], 网络链接配置文件

网络配置文件: vim /etc/sysconfig/network-scripts/ifcfg-ens33

网卡本身

网卡的连接配置
##nmcli

用法 描述
nmcli connection add help / nmcli c d h 查看 nmcli 帮助
nmcli device status 设备状况
nmcli device show ens33 / nmcli d show ens33 显示 ens33 状态
nmcli d disconnect ens33 / nmcli d d ens33 ens33 设备禁用
nmcli d connect ens33 / nmcli d c ens33 ens33 设备启用
nmcli connection show 显示连接
nmcli connection show ens33 显示ens33 网路连接
nmcli connection down ens33 / nmcli c d ens33 禁用连接配置
nmcli connection up ens33 / nmcli c u ens33 启用连接配置
nucli 参数 配置文件参数 描述
ipv4.method manual BOOTPROTO=none 静态分配 IP
ipv4.method auto BOOTPROTO=dhcp 动态分配 IP
ifname ens33 DEVICE=ens33 指定网卡名称
ipv4.addresses 192.168.100.100/24 IPADDR=192.168.100.100’/n’PREFIX=24 IP 地址与掩码位数
gw4 192.168.100.1(2?) GATEWAY=192.168.100.1 设置网关
ipv4.dns 8.8.8.8 DNS0=8.8.8.8 设置 DNS 服务器
ipv4.dns-search example.com DOMAIN=example.com 域名
connection.autoconnect yes ONBOOT=yes 开机自动连接

  • 创建名为 conn2 的连接配置, 指定静态 ip 地址, 不自动连接.
    nmcli c a ifname ens33 ipv4.method manual type Ethernet ipv4.addresses 192.168.85.136/24 gw4 192.168.85.2 autoconnect no con-name conn2

  • 经查看, 可知 conn2 连接已存在, 但未关联到某个设备上.
    nmcli c s

  • 查看 配置文件 conn2
    cat /etc/sysconfig/network-scripts/ifcfg-conn2

  • 首先禁用 ens33 连接, 再启用 conn2 连接. 这就是 两个网络情况下切换网络的方法.
    nmcli c d ens33
    nmcli c u conn2

  • 可通过查看连接状态了解详情.
    nmcli c s

  • 查看 ens33 设备的状态.
    nmcli d s
    nmcli d show ens33

  • 修改 conn2 的连接配置, IP为 , DNS 为 , 并设置开机自动启用连接.
    nmcli c m conn2 ipv4.addresses 192.168.85.59/24 ipv4.dns 192.168.85.1 connection.autoconnect yes

  • 查看 ifcfg-conn2配置文件的内容
    cat -n /etc/sysconfig/network-scripts/ifcfg-conn2

  • 重启网络服务, 使新配置生效.
    systemctl restart network

  • 将网络恢复原先状态
    nmcli c d conn2
