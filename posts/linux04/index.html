第四章 逻辑卷管理

创建于2022-06-09 Linux
post.toc

1、简介
LVM 是 Logical Volume Manager 的简称，中文就是逻辑卷管理。

1）物理卷（PV，Physical Volume）：就是真正的物理硬盘或分区。
2）卷组（VG，Volume Group）：将多个物理卷整合起来就组成了卷组，组成同一个卷组的物理卷可以是同一个硬盘的不同分区，也可以是不同硬盘上的不同分区。我们可以把卷组想象为一个逻辑硬盘。
3）逻辑卷（LV，Logical Volume）：卷组是一个逻辑硬盘，硬盘必须分区之后才能使用，这个分区我们称作逻辑卷。逻辑卷可以格式化和写入数据。我们可以把逻辑卷想象成为分区。
4）物理扩展（PE，Physical Extend）：PE 是用来保存数据的最小单元，我们的数据实际上都是写入 PE 当中，PE 的大小是可以配置的，默认是 4MB。

fdisk /dev/sda n… t 4 8e w # 硬盘分区，分区id设置为8e
pvcreate /dev/sda4 # 把物理分区建立物理卷
vgcreate -s 4MB vg_name /dev/sda4 /dev/sda5 # 把物理卷整合为卷组,-s PE 大小：指定 PE 的大小，单位可以是 MB,GB,TB 等。如果不写默认 PE 大小事 4MB
vgreduce vg_name /dev/sda5 # 添加减少卷组里的物理卷
lvcreate -L 512M -n lv_name vg_name # 卷组划分为逻辑卷,-L 容量：指定逻辑卷大小，单位 MB，GB，TB 等，-l 个数：按照 PE 个数指定逻辑卷大小，这个参数需要换算容量，太麻烦
lvresize -L +1M /dev/vg_koinl/lv_koinl # 调整逻辑卷的大小。lvdisplay’s LV Size: 513M ->大小改变了，但是 df -h /disklvm/‘s 文件系统：/dev/mapper/scvg-userlv 的容量：512M
resize2fs /dev/vg_koinl/lv_koinl #df -h /disklvm/‘s 文件系统：/dev/mapper/scvg-userlv 的容量：513M
# 挂载
mount /dev/vg_koinl/lv_koinl /挂载目录/ # 挂载目录需存在且已格式化。mkdir /挂载目录/ mkfs -t ext4 /dev/vg_koinl/lv_koinl

4.删除
umount /dev/vg_koinl/lv_koinl # 卸载挂载
e2fsck -f /dev/dd/yy # 卸载文件系统
lvremove /dev/vg_koinl/lv_koinl # 删除逻辑卷
vgremove vg_koinl # 删除卷组
pvremove /dev/sda4 # 删除物理卷
