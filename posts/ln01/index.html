10) chmod 命令: 修改权限

关于修改权限, 使用的是 chmod 777 file命令. 其中, 754 这三个数字分别表示自己, 所属组, 其他人的权限分别为 7, 5, 4. 数字分别代表的含义为: 1 表示可执行, 2 表示可读, 4 表示可写, 3, 5, 6, 7 分别表示他们的权重叠加.

关于权限问题, ll 长列表可以显示权限,
例如下 -rw-rw-r--. 中, - 表示普通文件, 第一个 rw- 表示自己可读可写不可执行, 第二个 rw- 表示同组可读可写不可执行, r-- 表示其他人可读不可写不可执行;
例若有 drwxr-xr-x. 中, d 表示目录文件, 第一个 rwx 表示自己可读可写可执行, 第二个 r-x 表示同组可读不可写可执行, 第三个 r-x 表示其他人可读不可写可执行;

linux第二次作业

点击显隐第二次

1.设置用户权限
（1）在主目录下创建目录test，进入目录test创建文件file，并且以长列表的形式显示文件的权限信息。
解: 创建目录: mkdir test, 进入目录: cd test, 创建文件: touch file, 长列表: ll.
（2）为文件file设置权限，使其他用户对此文件可以进行写操作。
解: 设置其他用户的写权限: chmod o+w file
（3）用数字设定法为file文件设置权限，所有者可读、可写、可执行，其他用户和所属组只有读和执行的权限。
解: 自己全操作, 组和他人读与执行: chmod 755 file
（4）用数字设定法更改file1文件的权限，使所有者只能读取文件，其他用户都没有权限。
解: 自己读, 其他无: chmod 400 file
（5）为其他用户添加写权限。
解: chmod o+w file
（6）回到上层目录，查看目录test的权限。
解: cd ../, ll
（7）为其他用户添加对此目录的写权限。
解: chmod o+w test
2.改变文件的所有者
（1）查看目录test及其文件的所属用户和用户组。
解: ll
（2）把目录test及其下所有文件的所有者改为om,所属组改为student。
解: root, chown -R om:student test
（3）删除目录test及其下的文件。
解; rm -r test

linux 第四次作业

点击显隐第四次

1.设置 at 调度, 要求在两天后上午 9 点创建目录 /test.
第一步: at 9am+2 days
第二步: mkdir /test
第三步: Ctrl + D 退出.

2.设置 cron 调度, 要求在每周一的 8 点执行关机操作.
第一步: crontab -e
第二步: 0 8 * * 1 shutdown -h now
3.请删除 /dev/sda 下的 sda2 分区.

第一步: 查看系统分区与挂载情况: lsblk -f
第二步: 卸载分区: umount /dev/sda2
第三步: fdisk /dev/sda.
第四步: d , 其中, d 删除, n添加, p 临时查看分区状态, q 退出不变更, w 保存.
第五步: 2, 删除第几个号
第六步: p , 可见 sdb2 没了
第七步: w , 保存更改
第八步: lsblk进行查看, 若fdisk已经消失, 但lsblk仍然有, 可以尝试partprobe /dev/sda刷新试试.

4.请创建一个大小为 300M 的主分区, 并将该分区挂在到 /mnt/data 目录下, 挂载完成后, 再对其进行卸载.
解: 第一步: fdisk /dev/sdb, 然后输入 n, 选择 p 为主分区 (e为扩展分区) , 输入数字 1, 表示 新建 sbd1;
First cylinder 默认设置直接回车; Last cylinder 则需要输入 +100M 表示这个分区大小为 100M, 最后输入 w 保存修改同时退出.
还是用命令 lsblk, lsblk -f, fdisk -l 查看分区状态.
第二步: 格式化分区, 命令是 mkfs -t ext4 /dev/sdb1. 使用 lsblk -f 查看分区情况, 这个时候 UUID 会出来, 但是没有挂载点 (MOUNTPOINT) .
第三步: 挂载, 挂载首先得设置路径, 没有的话使用 mkdir 创建.
然后使用 mount /dev/sdb2 /iflytek
然后用df -h 检查挂载情况
mkfs.vfat /dev/sdb1 (fat32分区)
mkfs.ext4 /dev/sdb1 (Linux分区)
mkfs.ntfs /dev/sdb1 (NTFS分区)

5.请创建一个大小为 1500M 的交换分区.
解: 第一步: 创建大小为 1500M 的交换文件: dd if=/dev/zero of=/swapfile bs=1024k count=1500
第二步: 使用命令 mkswap /swapfile 格式化交换文件
第三步: 使用命令 swapon /swapfile 启用交换分区
第四步: 使用命令 swapon -s 查看交换分区
